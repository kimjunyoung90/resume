기본 개념

토큰과 컨텍스트

토큰 (Token)

  • AI가 읽고 쓰는 단위
  • 영어는 짧은 단어 하나가 대략 1토큰
  • 한글은 같은 의미라도 더 많은 토큰 소모

컨텍스트 (Context)

  • AI가 답변 생성 시 참고할 수 있는 텍스트의 양
  • 대화 + 파일 + 설정 = 전부 컨텍스트에 포함
  • 가득 차면 대화 내용을 잊어버림
불필요한 정보가 컨텍스트를 차지할수록, AI 성능이 떨어진다.

컨텍스트가 길어지면?

Lost in the Middle

AI는 컨텍스트의 처음과 끝은 잘 잡지만, 중간에 있는 정보는 놓친다 (Stanford, 2023)

많이 넣는다고 좋은 게 아니라, 핵심만 넣어야 AI가 제대로 동작한다.

MCP — 주의사항

활성화된 MCP = 컨텍스트 소모

모든 MCP가 제공하는 기능 목록과 사용법이 컨텍스트에 로드됩니다.
예: Notion 15,076 토큰 + Slack 7,961 토큰 = MCP만으로 27,000+ 토큰

하지 말 것

사용하지 않는 MCP를 켜둔 채 작업

해야 할 것

미사용 MCP는 항상 꺼두기
필요한 기능만 담은 커스텀 MCP 사용

병렬 실행

순차

보안(2분) → 성능(2분) → 테스트(2분) = 총 6분

병렬

보안 + 성능 + 테스트 = 총 2분

Agent Teams

여러 Claude가 동시에 협업하는 팀
결제 모듈을 리팩토링할 에이전트 팀을 만들어줘.
- API 담당: 결제 API 엔드포인트와 DTO 리팩토링
- DB 담당: 결제 테이블 마이그레이션 스크립트 작성
- 테스트 담당: 결제 관련 통합 테스트 작성
서로 발견한 내용을 공유하면서 진행해.
혼자 다 안 해도 된다. 팀을 꾸려서 시키면 된다.

Hooks

특정 이벤트가 발생할 때 자동으로 실행할 동작을 정의
이벤트시점
PreToolUse도구 실행 전
PostToolUse도구 실행 후
Stop응답 완료 시
SessionStart세션 시작 시
UserPromptSubmit사용자 입력 시

응답 완료 시 알림 예시

{
  "hooks": {
    "Stop": [{
      "hooks": [{
        "type": "command",
        "command": "osascript -e '...'"
      }]
    }]
  }
}
생성 방법: 프롬프트로 요청하거나 settings.json에 직접 작성

Plugins

Skills + Hooks + MCP + Agents를 하나로 묶어 배포하는 패키지

구조

my-plugin/
├── .claude-plugin/
│   └── plugin.json   ← 매니페스트
├── skills/            ← 스킬들
├── agents/            ← 에이전트들
├── hooks/             ← 훅 설정
└── settings.json      ← 기본 설정

plugin.json

{
  "name": "my-plugin",
  "description": "팀 공통 개발 도구",
  "version": "1.0.0",
  "author": {
    "name": "Your Name"
  }
}
배포: Git repo → 마켓플레이스 등록 → /plugin install로 설치
